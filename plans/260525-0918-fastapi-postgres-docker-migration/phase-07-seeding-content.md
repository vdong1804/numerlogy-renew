# Phase 07 — Seeding & Content Loading

**Priority:** P1
**Effort:** S (4-6h)
**Status:** Done
**Depends on:** phase-02

## Goal

Seed initial data cho 22 numerology content tables + sample news + packages + banks + superuser. Để dev/staging có thể chạy ngay sau `docker compose up`.

## Data Sources

User chọn **fresh start** — không migrate từ MySQL prod. Có 2 lựa chọn:

### Option A: Export từ Django prod, transform
1. Dump 22 bảng content từ MySQL prod: `mysqldump --no-create-info --tables main_number mission_number ... > content.sql`
2. Transform sang Postgres-compatible SQL (escape quotes, fix `\r\n`)
3. Load qua psql

### Option B (Recommended): Python seed script
- Hardcode minimal content trong `scripts/seed_content.py`
- Admin team manually re-input thực content qua Next.js admin (phase 06)

→ **Decision: Option B** vì user chose fresh start. Seed chỉ enough để dev/test, prod content do admin tự nhập.

## Steps

1. Write `scripts/seed_content.py` — for each table, iterate codes, `INSERT ... ON CONFLICT (code) DO NOTHING`.
2. Add unique constraint trên `code` per content table (Alembic migration adjust).
3. Write `scripts/create_superuser.py` — argparse `--email --password`, hash password, create User với `is_superuser=True`, create UserProfile.
4. `scripts/seed_packages.py` — 3 packages: Free (0 downloads, 0 VND), Basic (3 downloads, 99k), Premium (10 downloads, 299k).
5. `scripts/seed_banks.py` — 1 sample bank (Vietcombank or as configured).
6. `scripts/seed_all.py` — orchestrate. Idempotent.
7. **Makefile target** `make seed` → `docker compose exec api python -m scripts.seed_all`.
8. **README** doc: how to run seed + create_superuser.

## Acceptance Criteria

- [x] `python -m scripts.seed_all` populates all 22 tables idempotently
- [x] Re-running doesn't duplicate rows (`ON CONFLICT DO NOTHING` works)
- [x] `python -m scripts.create_superuser --email admin@x.com --password Admin123!` creates user
- [x] Login với superuser → access `/admin/*` works
- [x] Sample `/api/so-hoc-free` request with seeded content returns valid PDF (even if content placeholder)

## Risks

- **Placeholder content** trong PDF không đẹp — chỉ ổn cho dev. Prod requires admin manual entry.
- **PhoneNumber rows** complex (`code__contains` lookup) — seed top 20-30 common patterns (`00`, `01`, ..., `99` first-2-digits + last-2-digits).
